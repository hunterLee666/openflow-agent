# Surprise Me

通过动态发现可用的技能并创造性地组合它们来提供意想不到的、令人愉悦的体验。

## 工作流程

### 步骤 1：发现可用的技能

阅读 <available_skills> 中列出的所有技能。

### 步骤 2：规划惊喜

选择 **1 到 3** 个技能并设计一个创意组合。目标是单一的内聚交付物，而不是单独的演示。

**创意组合原则：**
- 以意想不到的方式并列技能（例如，关于算法艺术的演示文稿，变成幻灯片的研究报告，带有 Canvas 设计插图的时尚文档）
- 如果有的话，结合用户已知的兴趣/上下文
- 优先考虑视觉冲击和情感愉悦，而不是信息密度
- 输出应该感觉像一份礼物——精致、令人惊讶且有趣

**主题创意（选择或混合）：**
- 与今天的日期、季节或热门新闻相关的内容
- 用户从未要求但会喜欢的迷你创意项目
- 一个有趣的"如果...会怎样"概念
- 结合数据+设计的审美产物
- 一个有趣的互动 HTML/React 体验

### 步骤 3：后备方案——没有其他可用技能

如果没有发现其他技能（只有 surprise-me 存在），使用以下后备方案之一：

1. **基于新闻的惊喜**：搜索今天的新闻找一个引人入胜的故事，然后创建一个精美的 HTML 产物以视觉 striking 的方式呈现
2. **互动 HTML 体验**：构建一个创意的单页网页体验——生成艺术、小游戏、视觉诗歌、动画信息图或互动故事
3. **个性化产物**：使用已知的用户上下文来创建个人化和令人愉悦的东西

### 步骤 4：执行

1. 阅读每个选定技能的完整 SKILL.md 正文
2. 按照每个技能的说明进行技术执行
3. 将输出组合成一个内聚的交付物
4. 用最少的开场白展示结果——让作品自己说话

### 步骤 5：揭示

用最少的剧透来展示惊喜。一句简短的预告，然后是产物。

- **好的揭示：** "我为你做了一些东西 ✨" + [产物]
- **不好的揭示：** "我决定将 pptx 技能与 canvas-design 技能结合起来，创建一个关于..."的演示文稿（破坏惊喜）
