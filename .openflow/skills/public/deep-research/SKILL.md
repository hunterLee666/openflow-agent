# 深度研究技能

## 概述

此技能提供进行彻底网络研究的系统方法。**在任何内容生成任务之前加载此技能**，以确保从多个角度、深度和来源收集足够的信息。

## 何时使用此技能

**当以下情况发生时，始终加载此技能：**

### 研究问题
- 用户问"什么是 X"、"解释 X"、"研究 X"、"调查 X"
- 用户想深入理解一个概念、技术或主题
- 问题需要来自多个来源的当前、全面信息
- 单个网络搜索不足以正确回答

### 内容生成（预研究）
- 创建演示文稿（PPT/幻灯片）
- 创建前端设计或 UI 模型
- 撰写文章、报告或文档
- 制作视频或多媒体内容
- 需要现实世界信息、示例或当前数据的任何内容

## 核心原则

**永远不要仅基于一般知识生成内容。** 你的输出质量直接取决于事先进行的研究的质量和数量。单次搜索查询永远不够。

## 研究方法论

### 阶段1：广泛探索

从广泛的搜索开始以了解整体情况：

1. **初步调查**：搜索主题以了解整体背景
2. **识别维度**：从初步结果中，识别需要更深入探索的关键子主题、主题或角度
3. **映射领域**：注意存在的不同观点、利益相关者或角度

示例：
```
主题："医疗保健中的 AI"
初步搜索：
- "AI 医疗应用 2026"
- "人工智能医学诊断"
- "医疗 AI 市场趋势"

识别的维度：
- 诊断 AI（放射学、病理学）
- 治疗推荐系统
- 行政自动化
- 患者监测
- 监管环境
- 伦理考虑
```

### 阶段2：深入研究

对于每个重要维度，进行针对性研究：

1. **具体查询**：为每个子主题使用精确关键词搜索
2. **多种措辞**：尝试不同的关键词组合和措辞
3. **获取完整内容**：使用 `web_fetch` 读取重要来源的完整内容，而不仅仅是片段
4. **追踪参考文献**：当来源提到其他重要资源时，也搜索这些

示例：
```
维度："放射学中的诊断 AI"
针对性搜索：
- "AI 放射学 FDA 批准系统"
- "胸部 X 光 AI 检测准确性"
- "放射学 AI 临床试验结果"

然后获取并阅读：
- 关键研究论文或摘要
- 行业报告
- 真实案例研究
```

### 阶段3：多样性与验证

通过寻求多样的信息类型确保全面覆盖：

| 信息类型 | 目的 | 示例搜索 |
|-----------------|---------|------------------|
| **事实与数据** | 具体证据 | "统计数据"、"数据"、"数字"、"市场规模" |
| **示例与案例** | 真实世界应用 | "案例研究"、"示例"、"实施" |
| **专家意见** | 权威观点 | "专家分析"、"访谈"、"评论" |
| **趋势与预测** | 未来方向 | "2026 年趋势"、"预测"、"未来" |
| **比较** | 背景和替代方案 | "vs"、"比较"、"替代方案" |
| **挑战与批评** | 平衡视角 | "挑战"、"局限性"、"批评" |

### 阶段4：综合检查

在进入内容生成之前，验证：

- [ ] 我是否从至少 3-5 个不同角度进行了搜索？
- [ ] 我是否获取并阅读了最重要来源的完整内容？
- [ ] 我是否有具体数据、示例和专家观点？
- [ ] 我是否探索了正面方面和挑战/局限性？
- [ ] 我的信息是否是最新的且来自权威来源？

**如果任何答案是否定的，在生成内容之前继续研究。**

## 搜索策略提示

### 有效的查询模式

```
# 带上下文的具体化
❌ "AI 趋势"
✅ "2026 年企业 AI 采用趋势"

# 包含权威来源提示
"[主题] 研究论文"
"[主题] 麦肯锡报告"
"[主题] 行业分析"

# 搜索特定内容类型
"[主题] 案例研究"
"[主题] 统计数据"
"[主题] 专家访谈"

# 使用时间限定符 — 始终使用 <current_date> 中的实际当前年份
"[主题] 2026"   # ← 用真实的当前年份替换，不要硬编码过去的年份
"[主题] 最新"
"[主题] 最新进展"
```

### 时间意识

**在形成任何搜索查询之前，始终检查 `<current_date>`。**

`<current_date>` 给出完整日期：年、月、日和星期几（例如 `2026-02-28, Saturday`）。根据用户询问的内容，使用适当的时间精度：

| 用户意图 | 需要的时间精度 | 示例查询 |
|---|---|---|
| "今天/今天早上/刚刚发布" | **月 + 日** | `"2026年2月28日 技术新闻"` |
| "本周" | **周范围** | `"2026年2月24日那一周 技术发布"` |
| "最近/最新/新" | **月** | `"2026年2月 AI 突破"` |
| "今年/趋势" | **年** | `"2026年 软件趋势"` |

**规则：**
- 当用户问关于"今天"或"刚刚发布"时，在搜索查询中使用**月 + 日 + 年**以获取当天结果
- 当需要日级别精度时，不要降到仅年份 — `"2026年技术新闻"` 不会显示今天的新闻
- 尝试多种措辞：数字形式（`2026-02-28`）、书面形式（`2026年2月28日`）和相对术语（`today`、`this week`）跨不同查询

❌ 用户问"今天科技有什么新消息" → 搜索 `"2026年新技术"` → 错过了今天的新闻
✅ 用户问"今天科技有什么新消息" → 搜索 `"2026年2月28日 新技术"` + `"2月28日 技术新闻"` → 获取今天的结果

### 何时使用 web_fetch

在以下情况下使用 `web_fetch` 读取完整内容：
- 搜索结果看起来高度相关和权威
- 你需要超出片段的详细信息
- 来源包含数据、案例研究或专家分析
- 你想理解一个发现的完整背景

### 迭代细化

研究是迭代的。初步搜索后：
1. 回顾你学到的东西
2. 识别你理解中的差距
3. 形成新的、更有针对性的查询
4. 重复直到你有全面覆盖

## 质量标准

你的研究足够当你能够自信地回答：
- 关键事实和数据点是什么？
- 2-3 个具体的真实世界示例是什么？
- 关于这个主题专家怎么说？
- 当前趋势和未来方向是什么？
- 挑战或局限性是什么？
- 什么使这个主题现在相关或重要？

## 应避免的常见错误

- ❌ 在 1-2 次搜索后停止
- ❌ 仅依赖搜索片段而不阅读完整来源
- ❌ 只搜索多面主题的一个方面
- ❌ 忽略相反的观点或挑战
- ❌ 当存在当前数据时使用过时的信息
- ❌ 在研究完成之前开始内容生成

## 输出

完成研究后，你应该有：
1. 从多个角度对主题的全面理解
2. 具体事实、数据点和统计数据
3. 真实世界的示例和案例研究
4. 专家观点和权威来源
5. 当前趋势和相关背景

**只有这样才进入内容生成**，使用收集的信息创建高质量、消息灵通的内容。
