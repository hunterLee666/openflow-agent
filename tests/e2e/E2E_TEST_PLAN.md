# OpenFlow E2E 测试方案

## 1. 测试概述

### 1.1 测试目标
- 验证 OpenFlow 系统前后端完整集成的正确性
- 确保核心功能在真实使用场景下正常工作
- 发现并修复端到端流程中的问题
- 验证系统在边缘案例下的稳定性
- 建立自动化回归测试套件

### 1.2 测试范围
- **后端核心系统**：OpenFlowCore、Bridge、Session管理
- **前端TUI**：用户交互、消息流、组件渲染
- **内存系统**：多层记忆、意图识别、语义检索
- **工具执行**：文件操作、Git操作、代码执行
- **技能系统**：内置技能、自定义技能、技能链
- **命令系统**：Slash命令、循环命令、Cron任务
- **协议通信**：WebSocket传输、消息序列化
- **错误处理**：异常恢复、用户反馈

## 3. 测试场景设计

### 3.1 真实用户场景（核心路径）

#### 场景 1: 首次启动与配置
```
用户流程：
1. 启动 OpenFlow 应用
2. 配置 API Key 和模型
3. 选择工作目录
4. 验证初始化完成

验证点：
- 配置正确保存
- 目录结构创建成功
- LLM连接验证通过
- 内存系统初始化完成
```

#### 场景 2: 代码编辑与工具使用
```
用户流程：
1. 发送查询："查看当前目录的文件"
2. 系统使用 FileList 工具列出文件
3. 用户："读取 package.json 的内容"
4. 系统使用 FileRead 工具并展示内容
5. 用户："创建一个 README.md 文件"
6. 系统使用 FileWrite 工具创建文件

验证点：
- 工具调用正确执行
- 文件操作成功完成
- 消息流正确渲染
- 上下文记忆保持
```

#### 场景 3: Git 操作与代码审查
```
用户流程：
1. 用户："查看当前Git状态"
2. 系统执行 git status
3. 用户："初始化Git仓库"
4. 系统执行 git init
5. 用户："使用代码审查技能检查最近的变更"
6. 系统调用 code_review 技能
7. 系统返回审查报告

验证点：
- Git工具正常工作
- 技能正确绑定和执行
- 审查报告正确生成
- 历史记录保存
```

#### 场景 4: 多轮对话与记忆
```
用户流程：
1. 用户："我在开发一个Todo应用"
2. 系统记录上下文
3. 用户："添加用户认证功能"
4. 系统关联之前的上下文
5. 用户："我们刚才在讨论什么项目？"
6. 系统从记忆中检索并回答

验证点：
- 上下文正确保持
- 记忆正确检索
- 多轮对话连贯性
- 意图识别准确
```

#### 场景 5: 技能链与复杂任务
```
用户流程：
1. 用户："重构这段代码，然后运行测试，如果测试通过就提交"
2. 系统：
   - 调用 refactor_assistant 技能
   - 执行测试命令
   - 调用 git 工具提交
3. 用户验证结果

验证点：
- 技能链正确编排
- 多步骤任务完成
- 状态管理正确
- 用户反馈清晰
```

#### 场景 6: Cron 任务管理
```
用户流程：
1. 用户："每天凌晨2点运行代码审查"
2. 系统创建Cron任务
3. 用户："查看所有计划任务"
4. 系统列出现有任务
5. 用户："暂停这个任务"
6. 系统暂停任务执行

验证点：
- Cron任务正确创建
- 任务列表正确显示
- 任务状态正确更新
- 持久化正确工作
```

#### 场景 7: Slash 命令与快捷操作
```
用户流程：
1. 用户输入 /help
2. 系统显示帮助信息
3. 用户输入 /undo
4. 系统撤销上一步操作
5. 用户输入 /compact
6. 系统压缩历史记录

验证点：
- 命令正确解析
- 命令功能正常
- 状态正确回滚
- 帮助信息完整
```

#### 场景 8: TUI 交互与导航
```
用户流程：
1. 用户使用 Vim 模式：ESC -> j -> k -> i
2. 用户输入文本
3. 用户使用 Tab 键自动补全
4. 用户使用 Ctrl+C 退出

验证点：
- Vim 模式正确响应
- 键盘快捷键工作
- 输入状态正确管理
- 界面渲染正确
```

## 4. 测试实现方案

### 4.1 测试工具类
```typescript
// 测试脚手架
class E2ETestHarness {
  tempDir: string
  mockLLM: MockLLM
  bridge: TestBridge
  core: OpenFlowCore
  
  async setup()
  async teardown()
  async sendQuery(message: string)
  async getState()
}
```

### 4.2 Mock 实现
- **MockLLM**：模拟 LLM API，返回预定义响应
- **MockFileSystem**：内存文件系统，支持读写操作
- **MockBridge**：测试用的通信桥
- **MockWorkspace**：模拟的工作区环境

### 4.3 测试数据生成
- 生成测试项目结构
- 创建模拟代码文件
- 预填充对话历史
- 配置测试场景

## 5. 测试执行计划

### 5.1 测试阶段
1. **基础设施搭建**：Mock、测试脚手架

4. **性能测试**：负载和压力测试
