# 新闻通讯生成技能

## 概述

此技能生成专业的、经过深入研究的新闻通讯，结合来自多个来源的策划内容和原创分析及评论。它遵循来自 Morning Brew、The Hustle、TLDR 和 Benedict Evans 等出版物的现代新闻通讯最佳实践，制作信息丰富、引人入胜且可操作的 内容。

输出是完整的、可以直接发布的 Markdown 格式新闻通讯，适用于电子邮件分发平台、网络发布或转换为 HTML。

## 核心能力

- 对指定主题进行来自多个网络来源的研究和内容策划
- 生成具有一致风格的主题聚焦或多主题新闻通讯
- 撰写引人入胜的标题、摘要和原创评论
- 构建最佳可读性和扫描性的内容结构
- 支持多种新闻通讯格式（每日摘要、周报、深度探讨、行业简报）
- 包含相关链接、来源和归属
- 针对目标受众调整语气和风格（技术型、高管型、一般型）
- 生成具有一致品牌和结构的定期新闻通讯系列

## 何时使用此技能

**当以下情况发生时，始终加载此技能：**

- 用户要求生成新闻通讯、电子邮件摘要或内容综述
- 用户请求策划关于某个主题的新闻或发展摘要
- 用户想要创建定期新闻通讯格式
- 用户要求将某个领域的最新发展编译成简报
- 用户需要包含多个策划项目的格式化、可直接发送电子邮件的内容
- 用户要求"周报"、"月度摘要"或"早间简报"

## 新闻通讯工作流程

### 阶段1：规划

#### 步骤1.1：了解新闻通讯需求

识别关键参数：

| 参数 | 描述 | 默认 |
|-----------|-------------|---------|
| **主题** | 要涵盖的主要主题领域 | 必填 |
| **格式** | 每日摘要、周报、深度探讨或行业简报 | 周报 |
| **目标受众** | 技术型、高管型、一般型或小众社区 | 一般型 |
| **基调** | 专业型、对话型、幽默型或分析型 | 对话型-专业型 |
| **长度** | 短（5分钟阅读）、中（10分钟）、长（15分钟+） | 中 |
| **章节** | 内容章节的数量和类型 | 4-6个章节 |
| **频率上下文** | 一次性还是定期系列的一部分 | 一次性 |

#### 步骤1.2：定义新闻通讯结构

根据格式，选择适当的结构：

**每日摘要结构**：
```
1. 头条故事（1条，详细）
2. 快讯（3-5条，简短）
3. 一个数据/今日引言
4. 值得关注
```

**周报结构**：
```
1. 编辑寄语/引言
2. 头条故事（2-3条，详细）
3. 趋势与分析（1-2条，原创评论）
4. 快讯（4-6条，简短摘要）
5. 工具与资源（2-3条）
6. 另一件事/结语
```

**深度探讨结构**：
```
1. 引言与背景
2. 背景/为什么重要
3. 关键发展（详细分析）
4. 专家观点
5. 下一步/影响
6. 进一步阅读
```

**行业简报结构**：
```
1. 执行摘要
2. 市场发展
3. 公司动态
4. 产品与技术更新
5. 监管与政策变化
6. 数据与指标
7. 展望
```

### 阶段2：研究与策划

#### 步骤2.1：多来源研究

使用网络搜索进行彻底研究。**新闻通讯的质量直接取决于研究的质量和新鲜度。**

**搜索策略**：

```
# 当前新闻和发展
"[主题] [当前月份] [当前年份] 新闻"
"[主题] 最新发展"
"[主题] 本周公告"

# 趋势和分析
"[主题] [当前年份] 趋势"
"[主题] 专家意见分析"
"[主题] 行业报告"

# 数据和统计
"[主题] [当前年份] 统计"
"[主题] 最新市场数据"
"[主题] 增长指标"

# 工具和资源
"[主题] [当前年份] 新工具"
"[主题] 开源发布"
"[主题] [当前年份] 最佳资源"
```

> **重要**：始终检查 `<current_date>` 以确保搜索查询使用正确的时间上下文。切勿使用硬编码的年份。

#### 步骤2.2：来源评估和选择

评估每个来源并策划最佳内容：

| 标准 | 优先级 |
|-----------|----------|
| **新鲜度** | 优先选择最近7-30天的内容 |
| **权威性** | 优先选择主要来源、官方公告、知名出版物 |
| **独特性** | 选择提供新视角或未被充分报道的故事 |
| **相关性** | 每个项目必须明确连接到新闻通讯声明的主题 |
| **可操作性** | 优先选择读者可以采取行动的内容（工具、见解、策略） |
| **多样性** | 新闻、分析、数据和实用资源的混合 |

#### 步骤2.3：深度内容提取

对于关键故事，使用 `web_fetch` 阅读完整文章并提取：

1. **核心事实** — 发生了什么，谁参与，何时
2. **背景** — 为什么这重要，背景信息
3. **数据点** — 具体数字、指标或统计数据
4. **引言** — 相关的专家引言或官方声明
5. **影响** — 这对读者意味着什么

### 阶段3：撰写

#### 步骤3.1：新闻通讯头部

每个新闻通讯以一致的头部开始：

```markdown
# [新闻通讯名称]

*[标语或描述] — [日期]*

---

[可选：关于内容的一句话预览]
```

#### 步骤3.2：章节撰写指南

**头条/精选项目**：
- **标题**：引人注目、清晰、以利益为导向（不是点击诱饵）
- **导语**：让读者关心的开场句（1-2句）
- **正文**：关键事实和背景（2-4段）
- **为什么重要**：连接到读者的世界（1段）
- **来源链接**：始终归属并链接到原始来源

**快讯/简短项目**：
- **格式**：粗体标题 + 2-3句摘要 + 来源链接
- **重点**：每个项目一个关键要点
- **效率**：读者应该无需点击即可获得基本见解

**分析/评论章节**：
- **风格**：新闻通讯关于趋势或发展的独特视角
- **结构**：观察 → 背景 → 影响 → （可选）可操作的要点
- **证据**：每个主张都有数据或来源信息支持

#### 步骤3.3：撰写标准

| 原则 | 实现 |
|-----------|---------------|
| **可扫描** | 使用标题、粗体文本、项目符号和短段落 |
| **引人入胜** | 从最有趣的角度开始，而不是按时间顺序 |
| **简洁** | 每个句子都值得保留——无情地删除填充内容 |
| **准确** | 每个事实都有来源，每个数字都经过验证 |
| **可归属** | 始终使用内联链接归功于原始来源 |
| **人性化** | 像知识渊博的朋友一样写作，而不是像新闻稿 |

**针对受众的语气校准**：

| 受众 | 语气 | 示例 |
|----------|------|---------|
| **技术型** | 精确，无需解释的行话，假设有专业知识 | "新 API 支持通过流量控制窗口进行背压处理的 gRPC 流式传输。" |
| **高管型** | 聚焦影响、底线、战略 | "此次收购让 X 公司在企业细分市场获得 40% 的市场份额，直接威胁现有 Y 公司的定价权。" |
| **一般型** | 易理解，有类比，解释概念 | "把它想象成一个数据通用翻译器——它让任何应用无需学习新语言就能与任何数据库通信。" |

### 阶段4：组装与完善

#### 步骤4.1：组装新闻通讯

按照选定的结构模板将所有章节组合成最终文档。

#### 步骤4.2：页脚

每个新闻通讯以以下内容结束：

```markdown
---

*[新闻通讯名称] 是 [描述]。*
*[如何订阅/分享/提供反馈]*

*来源：所有链接都以内联方式提供。本新闻通讯策划和总结公开可用的信息，并附有原创评论。*
```

#### 步骤4.3：质量检查清单

在最终确定之前，验证：

- [ ] **每个事实主张都有来源链接** — 没有无来源的断言
- [ ] **所有链接都有效** — 验证来自搜索结果的 URL
- [ ] **日期引用使用实际的当前日期** — 没有硬编码或假设的日期
- [ ] **内容是当前的** — 所有主要项目都在预期时间范围内
- [ ] **没有重复故事** — 每个项目只出现一次
- [ ] **格式一致** — 标题、项目符号、链接在整个文档中使用相同的样式
- [ ] **覆盖均衡** — 不被单一来源或视角主导
- [ ] **长度合适** — 符合指定的长度目标
- [ ] **开篇引人入胜** — 前2句话让读者想要继续
- [ ] **结尾清晰** — 新闻通讯以令人难忘或可操作的内容结束
- [ ] **已校对** — 没有拼写错误、格式损坏或不完整的句子

## 适配示例

### 科技新闻通讯
- Emoji 使用：✅ 适度（章节标题）
- 章节：头条故事、深度探讨、快讯、开源聚焦、开发者工具
- 语气：技术-对话型

### 商业/金融新闻通讯
- Emoji 使用：❌ 极少到无
- 章节：市场概览、交易流、公司动态、数据角、展望
- 语气：专业-分析型

### 行业特定新闻通讯
- Emoji 使用：适度
- 章节：监管更新、市场数据、创新观察、人事动态、活动
- 语气：专家-权威型

### 创意/营销新闻通讯
- Emoji 使用：✅ 大量
- 章节：活动聚焦、趋势观察、本周热门、我们喜欢的工具、灵感
- 语气：热情-专业型

## 输出处理

生成后：

- 将新闻通讯保存到 `/mnt/user-data/outputs/newsletter-{topic}-{date}.md`
- 使用 `present_files` 工具向用户展示新闻通讯
- 提供调整章节、语气、长度或重点领域的选项
- 如果用户想要 HTML 输出，请注意 Markdown 可以使用标准工具转换

## 注意事项

- 此技能与 `deep-research` 技能结合使用效果最佳，以获得全面的主题覆盖——需要深度分析的新闻通讯请同时加载两者
- 始终使用 `<current_date>` 来获取搜索和新闻通讯日期引用中的时间上下文
- 对于定期新闻通讯，建议保持一致的结构，以便读者形成预期
- 策划时，质量优于数量——5个优质项目优于15个平庸项目
- 正确归因所有内容——新闻通讯通过透明来源建立信任
- 避免总结读者无法访问的付费内容
- 如果用户提供了要包含的特定 URL 或文章，请将它们与你的策划发现一起纳入
- 新闻通讯应该在摘要中提供足够的价值，使读者即使不点击每个链接也能受益
