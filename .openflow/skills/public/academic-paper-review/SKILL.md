# 学术论文审阅技能

## 概述

此技能对学术论文和研究出版物进行结构化的同行评审质量分析。它遵循顶级会议（NeurIPS、ICML、ACL、Nature、IEEE）使用的既定学术审阅标准，提供严格、建设性和平衡的评估。

审阅涵盖**总结、优势、劣势、方法论评估、贡献评价、文献定位和可操作的建议**——所有内容都基于论文本身的证据。

## 核心能力

- 解析并理解来自上传PDF或获取URL的学术论文
- 按照顶级会议审阅模板生成结构化审阅
- 评估方法论的严谨性（实验设计、统计有效性、可重复性）
- 评估贡献的创新性和重要性
- 通过针对性文献搜索将工作置于更广泛的研究领域中
- 识别局限性、差距和潜在改进
- 生成详细审阅和简明执行摘要两种格式
- 支持任何科学领域（计算机科学、生物、物理、社会科学等）的论文

## 何时使用此技能

**当以下情况发生时，始终加载此技能：**

- 用户提供论文URL（arXiv、DOI、会议论文集、期刊链接）
- 用户上传研究论文或预印本的PDF
- 用户要求"审阅"、"分析"、"批评"、"评估"或"总结"研究论文
- 用户想了解研究的优势和劣势
- 用户要求对学术工作进行同行评审风格的评估
- 用户请求帮助准备会议或期刊提交的审阅

## 审阅方法论

### 阶段1：论文理解

在形成任何判断之前，仔细阅读并理解论文。

#### 步骤1.1：识别论文元数据

提取并记录：

| 字段 | 描述 |
|-------|-------------|
| **标题** | 完整论文标题 |
| **作者** | 作者列表及所属机构 |
| **发表场所/状态** | 出版场所、预印本服务器或提交状态 |
| **年份** | 出版或提交年份 |
| **领域** | 研究领域和子领域 |
| **论文类型** | 实证、理论、综述、立场论文、系统论文等 |

#### 步骤1.2：深度阅读

系统地阅读论文：

1. **摘要与引言** — 识别声称的贡献和动机
2. **相关工作** — 注意作者如何将他们的工作与现有技术进行定位
3. **方法论** — 详细理解提出的方法、模型或框架
4. **实验/结果** — 检查数据集、基线、指标和报告的结果
5. **讨论与局限性** — 记录任何自我识别的局限性
6. **结论** — 将声称的结论与实际提供的证据进行比较

#### 步骤1.3：关键主张提取

明确列出论文的主要主张：

```
主张1：[关于贡献或发现的具体主张]
证据：[论文中支持此主张的证据]
强度：[强/中/弱]

主张2：[...]
...
```

### 阶段2：批判性分析

#### 步骤2.1：文献背景搜索

使用网络搜索了解研究领域：

```
搜索查询：
- "[论文主题] 2026年最新技术"
- "[关键方法名] 比较基准"
- "[作者] 之前的工作 [主题]"
- "[具体技术] 局限性批评"
- "[研究领域] 最新进展综述"
```

对关键相关论文或综述使用 `web_fetch` 来理解这项工作的定位。

#### 步骤2.2：方法论评估

使用以下框架评估方法论：

| 标准 | 需问的问题 | 评分 |
|-----------|-----------------|--------|
| **合理性** | 方法在技术上是正确的吗？有没有逻辑缺陷？ | 1-5 |
| **创新性** | 什么是真正的新东西 vs. 增量改进？ | 1-5 |
| **可重复性** | 细节是否足以重复？代码/数据可用吗？ | 1-5 |
| **实验设计** | 基线是否公平？消融是否充分？数据集是否合适？ | 1-5 |
| **统计严谨性** | 结果是否具有统计显著性？报告误差线了吗？多次运行？ | 1-5 |
| **可扩展性** | 方法是否可扩展？讨论了计算成本吗？ | 1-5 |

#### 步骤2.3：贡献重要性评估

评估重要性级别：

| 级别 | 描述 | 标准 |
|-------|-------------|----------|
| **里程碑** | 根本性地改变领域 | 新范式、广泛适用的突破 |
| **重要** | 推进技术状态的强贡献 | 清晰改进加坚实证据 |
| **中等** | 有一些局限性的有用贡献 | 增量但有效的改进 |
| **边缘** | 相对于现有工作的最小进展 | 小收益、窄适用性 |
| **低于阈值** | 不符合发表标准 | 根本缺陷、证据不足 |

#### 步骤2.4：优势与劣势分析

对于每个优势或劣势，提供：
- **是什么**：具体观察
- **在哪里**：部分/图/表引用
- **为什么重要**：对论文主张或实用性的影响

### 阶段3：审阅综合

#### 步骤3.1：组装结构化审阅

使用以下模板生成最终审阅。

## 审阅输出模板

```markdown
# 论文审阅：[论文标题]

## 论文元数据
- **作者**：[作者列表]
- **发表场所**：[出版场所或预印本服务器]
- **年份**：[年份]
- **领域**：[研究领域]
- **论文类型**：[实证/理论/综述/系统/立场]

## 执行摘要

[2-3段总结论文的核心贡献、方法和主要发现。
首先说明你的整体评估：论文做得好的地方、不足之处，
以及贡献是否足以达到声称的发表场所/影响级别。]

## 贡献总结

1. [第一个声称的贡献 — 一句话]
2. [第二个声称的贡献 — 一句话]
3. [如有其他贡献]

## 优势

### S1：[简洁的优势标题]
[详细解释，引用论文中的具体部分、图或表。
解释为什么这是一个优势及其重要性。]

### S2：[简洁的优势标题]
[...]

### S3：[简洁的优势标题]
[...]

## 劣势

### W1：[简洁的劣势标题]
[详细解释和具体引用。解释此劣势对论文主张的影响。
建议如何解决。]

### W2：[简洁的劣势标题]
[...]

### W3：[简洁的劣势标题]
[...]

## 方法论评估

| 标准 | 评分 (1-5) | 评估 |
|-----------|:---:|------------|
| 合理性 | X | [简要理由] |
| 创新性 | X | [简要理由] |
| 可重复性 | X | [简要理由] |
| 实验设计 | X | [简要理由] |
| 统计严谨性 | X | [简要理由] |
| 可扩展性 | X | [简要理由] |

## 给作者的问题

1. [能够澄清疑虑或歧义的具体问题]
2. [关于方法论选择或替代方法的问题]
3. [关于通用性或实际适用性的问题]

## 次要问题

- [拼写错误、格式问题、不清楚的图、符号不一致]
- [缺失的应该引用的参考文献]
- [提高清晰度的建议]

## 文献定位

[这项工作与当前最新技术的关系如何？
是否引用了关键的相关工作？比较是否公平和全面？
缺少哪些重要的相关工作？]

## 建议

**整体评估**：[接收/弱接收/边缘/弱拒绝/拒绝]

**信心**：[高/中/低] — [信心水平的原因]

**贡献级别**：[里程碑/重要/中等/边缘/低于阈值]

### 可操作的改进建议
1. [具体的、建设性的建议]
2. [具体的、建设性的建议]
3. [具体的、建设性的建议]
```

## 审阅原则

### 建设性批评
- **总是建议如何修复** — 不要只是指出问题；要提出解决方案
- **该表扬就表扬** — 即使是有缺陷的论文，也要承认真正的贡献
- **要具体** — 引用具体的部分、公式、图和表
- **区分次要和主要** — 区分致命缺陷和可修复的问题

### 客观性标准
- ❌ "这篇论文写得不好"（模糊、无帮助）
- ✅ "第3.2节在引入符号X时没有正式定义，使得定理1的证明难以理解。建议在问题公式化之后添加符号表。"（具体、可操作）

### 伦理审阅实践
- 不要根据作者声誉或所属机构来否定工作
- 根据其自身优点评估工作
- 建设性地标记潜在的伦理问题（数据集中的偏见、双重用途的影响）
- 维护未发表工作的保密性

## 按论文类型的调整

| 论文类型 | 重点领域 |
|------------|-------------|
| **实证** | 实验设计、基线、统计显著性、消融、可重复性 |
| **理论** | 证明正确性、假设合理性、界的紧度、与实践的联系 |
| **综述** | 全面性、分类质量、对近期工作的覆盖、综合见解 |
| **系统** | 架构决策、可扩展性证据、真实世界部署、工程贡献 |
| **立场** | 论点一致性、主张证据、影响潜力、描述的公平性 |

## 应避免的常见陷阱

- ❌ 审阅你希望写的论文而不是提交的论文
- ❌ 要求范围不合理的新实验
- ❌ 因为没有解决不同的问题而惩罚论文
- ❌ 过度受写作质量而非技术贡献的影响
- ❌ 将没有与你的工作进行比较视为劣势
- ❌ 只提供总结而没有批判性分析

## 质量检查清单

在最终确定审阅之前，验证：

- [ ] 论文已完整阅读（不仅仅是摘要和引言）
- [ ] 所有主要主张都已识别并针对证据进行评估
- [ ] 至少提供3个优势和3个劣势，并有具体引用
- [ ] 方法论评估表完整，包含评分和理由
- [ ] 给作者的问题针对真正的歧义，而不是修辞性批评
- [ ] 进行了文献搜索以将贡献置于背景下
- [ ] 建议是可操作的和建设性的
- [ ] 整体评估与已识别的优势和劣势一致
- [ ] 审阅语气专业且尊重
- [ ] 次要问题与主要问题分开

## 输出格式

- 以 **Markdown** 格式输出完整审阅
- 在沙盒中工作时，将审阅保存到 `/mnt/user-data/outputs/review-{paper-topic}.md`
- 使用 `present_files` 工具向用户展示审阅

## 注意事项

- 此技能补充 `deep-research` 技能 — 当用户想在更广泛领域的背景下审阅论文时，加载两者
- 对于付费墙后的论文，使用可访问的内容（摘要、公开可用版本、预印本镜像）
- 根据用户需求调整审阅深度：快速分类的简要评估 vs. 提交准备的全面的审阅
- 当比较审阅多篇论文时，在所有审阅中保持一致的标准
- 始终披露审阅的局限性（例如，"我无法详细验证附录B中的证明"）
