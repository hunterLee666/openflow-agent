# OpenFlow 终端 TUI E2E 测试方案 V2.0

## 1. 测试概述

### 1.1 测试目标
- 模拟真实用户从启动到退出的完整终端操作流程
- 验证所有键盘交互、输入、渲染、导航功能的正确性
- 覆盖正常流程、异常流程、边界场景
- 确保系统在各种条件下的稳定性和可靠性

### 1.2 测试范围
- **启动与初始化**：冷启动、热启动、连接状态
- **键盘输入**：字符输入、光标移动、删除、提交
- **快捷键**：Ctrl+C、Ctrl+L、Ctrl+R、Escape、Tab、箭头键
- **Vim 模式**：模式切换、命令执行、状态指示
- **消息流**：消息发送、接收、渲染、滚动
- **状态栏**：信息展示、实时更新
- **通知系统**：提示显示、自动消失
- **帮助系统**：帮助面板打开/关闭
- **退出流程**：双击退出、优雅退出
- **终端适配**：不同尺寸、不同终端类型
- **边界场景**：长文本、特殊字符、空输入、快速操作
- **并发操作**：多任务同时执行
- **错误恢复**：错误处理、状态恢复

## 3. 测试场景设计

### 3.1 启动与初始化场景

#### 场景 1: 冷启动
```
用户流程：
1. 执行启动命令
2. 等待应用初始化
3. 观察欢迎界面

验证点：
- 标题 "OpenFlow CLI" 正确显示
- 副标题正确显示
- 欢迎信息 "Welcome to OpenFlow CLI" 显示
- 输入提示 "> Type a message..." 显示
- 状态栏显示 "mode: chat", "connection: connected", "tokens: 0"
- 光标正确显示在输入位置
- 无黑屏、无渲染错误
```

#### 场景 2: 连接中状态
```
用户流程：
1. 启动应用（后端未就绪）
2. 观察加载状态
3. 等待连接成功

验证点：
- Spinner 组件显示 "thinking..."
- 连接状态从 "connecting" 变为 "connected"
- 加载完成后输入框可用
```

#### 场景 3: 连接失败
```

1. 启动应用（后端不可用）

验证点：
- 错误信息正确显示
- 错误区域背景色正确（#3a1a1a）
- 用户仍可输入（前端不崩溃）
```

## 4. 测试实现方案

### 4.1 测试工具类
```typescript
// 终端渲染测试工具
class TerminalRenderHarness {
  // 捕获 Ink 输出
  captureOutput(): string
  
  // 模拟键盘输入
  simulateKey(key: string): void
  
  // 验证渲染内容
  assertContains(text: string): void
  
  // 验证组件状态
  assertState(state: object): void
}

// 键盘事件模拟器
class KeyboardSimulator {
  // 模拟单个按键
  press(key: string): void
  
  // 模拟组合键
  pressCombo(keys: string[]): void
  
  // 模拟按键序列
  pressSequence(keys: string[]): void
}
```

### 4.2 测试数据结构
```typescript
interface TerminalTestCase {
  name: string
  description: string
  setup: () => Promise<void>
  actions: Array<{
    type: 'key' | 'text' | 'wait'
    value: string
    delay?: number
  }>
  assertions: Array<{
    type: 'contains' | 'notContains' | 'state' | 'render'
    value: string | object
  }>
  teardown: () => Promise<void>
}
```
