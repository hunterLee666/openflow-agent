# 专业研究报告技能

## 概述

此技能以 Markdown 格式生成专业、咨询级的研究报告，涵盖**市场分析、消费者洞察、品牌战略、财务分析、行业研究、竞争情报、投资研究和宏观经济分析**等领域。它分两个不同阶段运行：

1. **阶段1 — 分析框架生成**：给定研究主题，产生严谨的分析框架，包括章节骨架、每章数据需求、分析逻辑和可视化计划。
2. **阶段2 — 报告生成**：在数据被其他技能收集后，将所有输入综合成最终完善的报告。

输出符合麦肯锡/BCG 咨询 voice 标准。报告语言遵循 `output_locale` 设置（默认：`zh_CN` 为中文）。

## 数据真实性协议

**严格遵守规则**：报告中呈现的所有数据和图表中可视化的所有数据必须直接源自提供的**数据摘要**或**外部搜索结果**。
- **不 hallucinate**：不发明、估计或模拟数据。如果数据缺失，声明"数据不可用"，而不是编造数字。
- **可追溯来源**：每个主要主张和图表都必须能够追溯到输入数据包。

## 核心能力

- **从头设计分析框架**，仅给定研究主题和范围
- 将原始数据转换为结构化、高深度的研究报告
- 遵循每个子章节的 **"视觉锚点 → 数据对比 → 综合分析"** 流程
- 按照 **"数据 → 用户心理学 → 策略含义"** 链条产生见解
- 嵌入预生成的图表并构建对比表格
- 生成符合 **GB/T 7714-2015** 标准的内联引用
- 使用 `output_locale` 指定的语言输出报告，保持专业咨询语气
- 适应领域（营销、金融、行业等）的分析深度和结构

## 何时使用此技能

**当以下情况发生时，始终加载此技能：**

- 用户要求市场分析、消费者洞察报告、财务分析、行业研究或任何咨询级分析报告
- 用户提供研究主题，需要在数据收集之前提供结构化分析框架
- 用户提供数据摘要、分析框架或图表文件以综合成报告
- 用户需要专业咨询风格的研究报告
- 任务涉及将研究发现转化为结构化战略叙事

# 阶段1：分析框架生成

## 目的

给定**研究主题**（例如"Z世代护肤市场分析"、"新能源汽车行业竞争格局"、"品牌X消费者画像"），产生一个完整的**分析框架**，作为下游数据收集和最终报告生成的蓝图。

## 阶段1 输入

| 输入 | 描述 | 必填 |
|-------|-------------|----------|
| **研究主题** | 要分析的主题或问题 | 是 |
| **范围/约束** | 地理范围、时间范围、行业细分、目标受众等 | 可选 |
| **特定角度** | 用户想要探索的任何特定角度或假设 | 可选 |
| **领域** | 分析领域：市场、金融、行业、品牌、消费者、投资等 | 推断 |

## 阶段1 工作流程

### 步骤1.1：理解研究主题

- 解析研究主题以识别**核心实体**（市场、品牌、产品、行业、消费者群体、金融工具等）
- 识别**分析领域**（营销、金融、行业、竞争、消费者、投资、宏观等）
- 基于领域确定**自然分析维度**：

| 领域 | 典型维度 |
|--------|--------------------|
| 市场分析 | 市场规模、增长趋势、市场细分、增长驱动因素、竞争格局、消费者画像 |
| 品牌分析 | 品牌定位、市场份额、消费者认知、营销策略、竞争对手比较 |
| 消费者洞察 | 人口统计学画像、购买行为、决策旅程、痛点、场景分析 |
| 财务分析 | 宏观环境、行业趋势、公司基本面、财务指标、估值、风险评估 |
| 行业研究 | 价值链分析、市场规模、竞争格局、政策环境、技术趋势、进入壁垒 |
| 投资尽职调查 | 商业模式、财务健康、管理评估、市场机会、风险因素、退出途径 |
| 竞争情报 | 竞争对手识别、战略比较、SWOT分析、差异化定位、市场动态 |

### 步骤1.2：选择分析框架和模型

基于识别的领域和研究主题，选择**一个或多个**专业分析框架来构建每个章节的推理结构。选定的框架指导章节骨架中**步骤1.3的分析逻辑**。

#### 战略与环境分析

| 框架 | 描述 | 适用于 |
|-----------|-------------|----------|
| **SWOT 分析** | 优势、劣势、机会、威胁 | 品牌评估、竞争定位、战略规划 |
| **PEST/PESTEL 分析** | 政治、经济、社会、技术（+环境、法律） | 宏观环境扫描、市场进入评估、政策影响分析 |
| **波特五力** | 供应商议价能力、买方议价能力、新进入者威胁、替代品威胁、行业竞争 | 行业竞争格局、进入壁垒评估、利润率分析 |
| **波特钻石模型** | 要素条件、需求条件、相关产业、公司战略与结构 | 国家/地区竞争优势分析 |
| **VRIO 分析** | 价值、稀缺性、模仿性、组织 | 核心能力评估、资源优势分析 |

#### 市场与增长分析

| 框架 | 描述 | 适用于 |
|-----------|-------------|----------|
| **STP 分析** | 细分、目标、定位 | 市场细分、目标市场选择、品牌定位 |
| **BCG 矩阵（增长-份额矩阵）** | 明星、现金牛、问题产品、瘦狗 | 产品组合管理、资源分配决策 |
| **安索夫矩阵** | 市场渗透、市场开发、产品开发、多元化 | 增长战略选择 |
| **产品生命周期（PLC）** | 引入、成长、成熟、衰退 | 产品战略制定、市场时机决策 |
| **TAM-SAM-SOM** | 总可服务/可服务/可获得市场 | 市场规模量化、机会量化 |
| **技术采用生命周期** | 创新者 → 早期采用者 → 早期大众 → 晚期大众 → 落后者 | 新兴技术/类别渗透分析 |

#### 消费者与行为分析

| 框架 | 描述 | 适用于 |
|-----------|-------------|----------|
| **消费者决策旅程** | 认知 → 考虑 → 评估 → 购买 → 忠诚 | 消费者行为路径映射、触点优化 |
| **AARRR 漏斗（海盗指标）** | 获取、激活、留存、收入、推荐 | 用户增长分析、转化率优化 |
| **RFM 模型** | 最近购买时间、购买频率、消费金额 | 客户价值细分、精准营销 |
| **马斯洛需求层次** | 生理 → 安全 → 社交 → 尊重 → 自我实现 | 消费者心理学分析、产品价值主张 |
| **待完成工作（JTBD）** | 用户在特定背景下需要完成的"工作" | 需求洞察、产品创新方向 |

#### 财务与估值分析

| 框架 | 描述 | 适用于 |
|-----------|-------------|----------|
| **杜邦分析** | ROE = 净利润率 × 资产周转率 × 权益乘数 | 盈利能力分解、财务健康诊断 |
| **DCF（折现现金流）** | 自由现金流折现 | 企业/项目估值 |
| **可比公司分析** | 市盈率、市净率、市销率、EV/EBITDA 倍数比较 | 相对估值、同行基准 |
| **EVA（经济增加值）** | 税后营业利润 - 资本成本 | 价值创造能力评估 |

#### 竞争与战略定位

| 框架 | 描述 | 适用于 |
|-----------|-------------|----------|
| **基准分析** | 关键绩效指标逐项比较 | 竞争对手差距分析、最佳实践识别 |
| **战略群体映射** | 沿两个关键维度聚类竞争对手 | 竞争格局可视化、空白地带识别 |
| **价值链分析** | 主要活动 + 支持活动价值分解 | 成本优势来源、差异化机会识别 |
| **蓝海战略** | 价值曲线、四行动框架（消除-减少-提高-创造） | 差异化创新、新市场空间创造 |
| **感知映射** | 沿两个消费者感知维度绘制品牌位置 | 品牌定位分析、市场空白发现 |

#### 行业与供应链分析

| 框架 | 描述 | 适用于 |
|-----------|-------------|----------|
| **行业价值链** | 上游 → 中游 → 下游分解 | 行业结构理解、利润分布分析 |
| **Gartner 炒作周期** | 技术触发 → 期望峰值 → 泡沫化低谷 → 启蒙斜坡 → 生产力高原 | 新兴技术成熟度评估 |
| **GE-麦肯锡矩阵** | 行业吸引力 × 竞争优势 | 业务组合优先级、投资决策 |

#### 选择原则

1. **领域优先**：基于步骤1.1识别的领域，从上面的工具箱中选择**2-4个**最相关的框架
2. **互补性**：选择互补而不是重叠的框架（例如，宏观层面的 PESTEL + 微观层面的波特五力）
3. **深度优于广度**：深入应用2个框架比肤浅地堆叠6个更好
4. **数据可行性**：所选框架必须能够被下游数据收集技能支持 — 如果框架所需的数据无法合理获得，降级或替换
5. **明确映射**：在章节骨架中，明确注释每个章节使用哪个框架及其应用方式

#### 框架选择输出格式

```markdown
## 框架选择

| 章节 | 选择的框架 | 应用 |
|---------|----------------------|-------------|
| 市场规模与增长趋势 | TAM-SAM-SOM + 产品生命周期 | TAM-SAM-SOM 量化市场空间，PLC 确定市场阶段 |
| 竞争格局评估 | 波特五力 + 战略群体映射 | 五力评估行业竞争强度，群体映射可视化竞争定位 |
| 消费者画像 | RFM + 消费者决策旅程 | RFM 细分客户价值，决策旅程识别关键转化节点 |
| 品牌战略建议 | SWOT + 蓝海战略 | SWOT 总结整体格局，蓝海指导差异化方向 |
```

### 步骤1.3：设计章节骨架

产生分层章节结构。每个章节必须包括：

1. **章节标题** — 专业的、简洁的、基于主题的（遵循格式部分的标题约束）
2. **分析目标** — 本章旨在揭示什么
3. **分析逻辑** — 推理链或框架（必须引用步骤1.2中选择的框架）
4. **核心假设** — 需要被数据验证或反驳的初步假设

#### 章节骨架输出格式

```markdown
## 分析框架

### 章节1：[标题]
- **分析目标**：[本章旨在...]
- **分析逻辑**：[使用的框架或推理链]
- **核心假设**：[要验证的假设]
- **数据需求**：（见步骤1.4）
- **可视化计划**：（见步骤1.5）

### 章节2：[标题]
...
```

### 步骤1.4：定义每章的数据查询需求

对于每个章节，具体指定**需要收集的确切数据**。这是通往下游数据收集技能的桥梁。

每个数据需求条目必须包括：

| 字段 | 描述 |
|-------|-------------|
| **数据指标** | 所需的具体指标或数据点（例如，"2020-2025年中国护肤品市场规模（单位：十亿元人民币）"） |
| **数据类型** | 定量、定性或混合 |
| **建议来源** | 建议的来源类别：行业报告、财务报表、政府统计、社交媒体、电子商务平台、调查数据、新闻 |
| **搜索关键词** | 数据收集代理的建议搜索查询 |
| **优先级** | P0（必需）/ P1（重要）/ P2（补充） |
| **时间范围** | 数据应覆盖的时间段 |

#### 数据需求输出格式（每章）

```markdown
#### 数据需求

| # | 数据指标 | 数据类型 | 建议来源 | 搜索关键词 | 优先级 | 时间范围 |
|---|-------------|-----------|-------------------|-----------------|----------|------------|
| 1 | 市场规模（十亿元人民币） | 定量 | 行业报告、政府统计 | "中国护肤品市场规模 2024" | P0 | 2020-2025 |
| 2 | 复合年增长率 | 定量 | 行业报告 | "护肤品复合年增长率" | P0 | 2020-2025 |
| 3 | 子类别份额 | 定量 | 电子商务平台、行业报告 | "护肤品类别份额 乳霜 精华 防晒" | P1 | 最新 |
| 4 | 政策与监管更新 | 定性 | 政府公告、新闻 | "化妆品法规 2024" | P2 | 过去1年 |
```

### 步骤1.5：定义每章的可视化和内容结构

对于每个章节，指定**计划的可视化**和**最终报告的内容结构**：

| 字段 | 描述 |
|-------|-------------|
| **可视化类型** | 图表类型：折线图、柱状图、饼图、散点图、雷达图、热力图、桑基图、对比表格等 |
| **可视化标题** | 图表的描述性标题 |
| **可视化数据映射** | 哪些数据指标映射到 X/Y 轴或细分 |
| **对比表格设计** | 列标题和对比维度 |
| **论证结构** | 计划的"是什么 → 为什么 → 所以什么"叙事大纲 |

#### 可视化计划输出格式（每章）

```markdown
#### 可视化与内容计划

**图表1**：[类型] — [标题]
- X轴：[维度]，Y轴：[指标]
- 数据来源：对应数据需求 #1、#2

**对比表格**：
| 维度 | 项目A | 项目B | 项目C |
|-----------|--------|--------|--------|

**论证结构**：
1. **观察（是什么）**：[数据揭示的表面现象]
2. **归因（为什么）**：[驱动因素或根本原因]
3. **含义（所以什么）**：[战略含义或建议行动]
```

# 阶段1→2 交接：数据收集与图表生成

分析框架生成后，交给**其他数据收集技能**（例如 deep-research、data-analysis、网络搜索代理）：

1. 执行每章数据需求中的**搜索关键词**
2. 收集定量数据、定性见解和来源 URL
3. 基于**可视化和内容计划**生成图表
4. 返回包含以下内容的**数据包**：
   - **数据摘要**：每章的原始数字、指标和定性发现
   - **图表文件**：生成的图表图像及本地文件路径
   - **外部搜索结果**：引用的来源 URL 和摘要

> **此技能不执行数据收集。** 它仅生成框架（阶段1）和最终报告（阶段2）。
>
> **图表生成**：如果有可视化/图表技能可用（例如 data-analysis、image-generation），图表生成可推迟到阶段2开始 — 见步骤2.3。

# 阶段2：报告生成

## 目的

接收上游提供的完整**分析框架**和**数据包**，并将其综合成最终咨询级报告。

## 阶段2 输入

| 输入 | 描述 | 必填 |
|-------|-------------|----------|
| **分析框架** | 阶段1生成框架文档 | 是 |
| **数据摘要** | 数据收集阶段按章节组织收集的数据 | 是 |
| **图表文件** | 生成图表图像的本地文件路径。如未提供，将在步骤2.3使用可用可视化技能生成 | 可选 |
