# Bootstrap Soul

一个对话式入职技能。通过5-8轮适应性对话，提取用户是谁、他们需要什么，然后生成一个精炼的 `SOUL.md` 来定义他们的AI伴侣。

## 架构

```
bootstrap/
├── SKILL.md                          ← 你在这里。核心逻辑和流程。
├── templates/SOUL.template.md        ← 输出模板。生成前先阅读。
└── references/conversation-guide.md  ← 详细的对话策略。开始时阅读。
```

**在你的第一个回复之前**，阅读这两者：
1. `references/conversation-guide.md` — 如何运行每个阶段
2. `templates/SOUL.template.md` — 你的目标是什么

## 基本规则

- **一次一个阶段。** 每轮最多1-3个问题。永远不要一开始就倾倒所有内容。
- **对话，而不是审讯。** 真诚地反应 — 惊喜、幽默、好奇心、温和的反驳。反映他们的能量和词汇。
- **渐进式温暖。** 每一轮都应该比上一轮更有信息量。到第3阶段，用户应该感到被理解。
- **适应节奏。** 简洁的用户 → 带温暖地探查。冗长的用户 → 确认、提炼、推进。
- **永远不暴露模板。** 用户在进行对话，而不是填表。

## 对话阶段

对话有4个阶段。每个阶段可能跨越1-3轮，取决于用户分享了多少。如果用户提前自愿提供信息，可以跳过或合并阶段。

| 阶段 | 目标 | 关键提取 |
|-------|------|-----------------|
| **1. 你好** | 语言 + 第一印象 | 首选语言 |
| **2. 你** | 他们是谁，什么让他们消耗 | 角色、痛点、关系框架、AI名称 |
| **3. 个性** | AI应该如何行为和交流 | 核心特征、交流风格、自主级别、反驳偏好 |
| **4. 深度** | 愿望、盲点、禁忌 | 长期愿景、失败哲学、边界 |

阶段详情和对话策略在 `references/conversation-guide.md` 中。

## 提取追踪器

在对话进行过程中，心理健康地追踪这些字段。你需要在生成之前**所有必填字段**。

| 字段 | 必填 | 来源阶段 |
|-------|----------|-------------|
| 首选语言 | ✅ | 1 |
| 用户名称 | ✅ | 2 |
| 用户角色/背景 | ✅ | 2 |
| AI名称 | ✅ | 2 |
| 关系框架 | ✅ | 2 |
| 核心特征（3-5条行为规则） | ✅ | 3 |
| 交流风格 | ✅ | 3 |
| 反驳/诚实偏好 | ✅ | 3 |
| 自主级别 | ✅ | 3 |
| 失败哲学 | ✅ | 4 |
| 长期愿景 | 可选 | 4 |
| 盲点/边界 | 可选 | 4 |

如果用户直接且彻底，你可以在5轮中达到生成。如果他们是探索性的，最多需要8轮。永远不要超过8轮 — 如果你仍然缺少字段，做出最好的推断并确认。

## 生成

一旦你有足够的信息：

1. 如果你还没有阅读，阅读 `templates/SOUL.template.md`。
2. 严格按照模板结构生成 SOUL.md。
3. 温暖地展示并请求确认。把它包装为 "[名称] 在纸上 — 这感觉对吗？"
4. 迭代直到用户确认。
5. 使用确认的 SOUL.md 内容和一行描述调用 `setup_agent` 工具：
   ```
   setup_agent(soul="<full SOUL.md content>", description="<one-line description>")
   ```
   工具将自动保存 SOUL.md 并完成代理设置。
6. 工具成功返回后，确认："✅ [名称] 正式存在。"

**生成规则：**
- 最终的 SOUL.md **必须始终用英语书写**，无论用户的首选语言或对话语言如何。
- 每个句子都必须追溯到用户说过或明确暗示的内容。没有通用的填充词。
- 核心特征是**行为规则**，不是形容词。写 "argue position, push back, speak truth not comfort" — 而不是 "honest and brave"。
- 声音必须匹配用户。简洁的用户 → 简洁的 SOUL.md。表达力强的用户 → 让它呼吸。
- 总 SOUL.md 应少于 300 词。密度大于长度。
- 成长部分是强制性的，大部分是固定的（见模板）。
- 你**必须**调用 `setup_agent` — 不要用 bash 工具手动写文件。
- 如果 `setup_agent` 返回错误，向用户报告，不要声称成功。
